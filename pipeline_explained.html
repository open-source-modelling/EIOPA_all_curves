Code walkthrough

How the EIOPA RFR extraction pipeline works

A step-by-step explanation of each file in the pipeline: what it reads, what it does to it, and why it's built the way it is. Written against the November 2024 EIOPA risk-free rate files.

config.py extractors/qb_extractor.py extractors/param_extractor.py csv_store.py main.py

Overview

EIOPA publishes two workbooks every month: one with the Smith‑Wilson Qb calibration vectors, one with the resulting term structures and the parameters used to build them. Both follow the same filename pattern and both repeat the same per‑country block layout across dozens of columns — good for a human reading it in Excel, awkward for anything downstream. The pipeline turns both into flat CSV tables that keep growing, one month at a time.

Qb_SW.xlsx
Smith‑Wilson Qb vectors, 2 sheets (no‑VA / with‑VA)
→
qb_extractor.py
finds each country's block, walks it down to a long table
→
qb_vectors.csv
date · curve · country · term · value
Term_Structures.xlsx
RFR_spot sheets, 6 fixed parameter rows per country
→
param_extractor.py
reads the 7 parameter rows for every country column
→
curve_parameters.csv
date · curve · country · alpha, UFR, CRA, VA…

main.py ties both extractors together, pulls the reference date out of the filename, and hands the result to csv_store.py, which merges it into the running CSVs without creating duplicates if you run the same month twice.

01

Reading the filename

config.py

EIOPA's own filenames already tell you the reference date: EIOPA_RFR_20241130_Qb_SW.xlsx. Rather than ask the user to type the date in separately (and risk it not matching the file), config.py pulls it straight out with a regex:

FILENAME_DATE_RE = re.compile(r"EIOPA_RFR_(\d{8})_")

def extract_reference_date(filename):
    m = FILENAME_DATE_RE.search(Path(filename).name)
    if not m:
        raise ValueError(f"no EIOPA_RFR_YYYYMMDD_ stamp in {filename}")
    raw = m.group(1)
    return f"{raw[0:4]}-{raw[4:6]}-{raw[6:8]}"  # -> "2024-11-30"

If a file doesn't match the pattern — wrong name, wrong folder, a stray copy — this raises immediately instead of silently tagging the data with the wrong month. This module also just holds the shared output paths (data/qb_vectors.csv, data/curve_parameters.csv) so every other file agrees on where things live.

02

Extracting the Qb vectors

extractors/qb_extractor.py

The workbook has two sheets, SW_Qb_no_VA and SW_Qb_with_VA. Inside each, one header row holds a run of country codes (EUR, AT, BE, BG …), and every country owns a 3‑column block: a term‑index column, a value column, and a blank spacer column.

ColCol+1Col+2
1, 2, 3 … (term index)‑11.43, 8.10 … (Qb value)blank spacer

Step by step, the extractor:

  1. Finds the header row by scanning the first 25 rows for the one where more than a handful of cells hold short text labels (2–4 characters) — that's the row of country codes, wherever EIOPA happens to have put it that month.
  2. Maps every country to its value column by re‑reading that header row: any cell holding text is a country, and its column number is the "value column" for that block.
  3. Walks down each column starting the row after the header, reading (term index, value) pairs until it hits a row where both are empty — that's the end of that country's data.
  4. Stacks every country, from both sheets, into one long table with columns reference_date, curve_type, country, term_index, qb_value.
def _extract_sheet(ws, curve_type, reference_date):
    header_row = _find_header_row(ws)
    country_cols = {c: ws.cell(row=header_row, column=c).value
                     for c in range(1, ws.max_column + 1)
                     if isinstance(ws.cell(row=header_row, column=c).value, str)}

    for value_col, country in country_cols.items():
        index_col = value_col - 1
        r = header_row + 1
        while True:
            term_index = ws.cell(row=r, column=index_col).value
            qb_value   = ws.cell(row=r, column=value_col).value
            if term_index is None and qb_value is None:
                break
            # ...append the row, r += 1
Design choice — term_index is kept as a float, not an int. Most countries index by whole years (1, 2, 3…), but Australia, Canada, China, Hong Kong, Mexico, New Zealand, Singapore, South Africa, South Korea and Thailand step in half‑years (0.5, 1, 1.5, 2…) because of their coupon frequency. Rounding to int would collide 0.5 and 1.0 into the same row.
03

Extracting the curve parameters

extractors/param_extractor.py

The Term_Structures.xlsx workbook lays out its RFR_spot_no_VA / RFR_spot_with_VA sheets differently — no repeating vector here, just six or seven fixed parameter rows sitting above each country's spot‑rate curve:

RowLabelMeaning
3AT_30_11_2024_SWP_LLP_20_...country code + instrument type, encoded in one string
4Coupon_freqcoupon frequency assumption
5LLPlast liquid point (years)
6Convergenceyears to converge to the UFR
7UFRultimate forward rate
8alphaSmith‑Wilson convergence speed parameter
9CRAcredit risk adjustment (bps)
10VAvolatility adjustment (bps) — populated only on the with‑VA sheet

Row 3 is the interesting one: EIOPA packs the country code and the underlying instrument type into a single string, e.g. AT_30_11_2024_SWP_LLP_20_EXT_40_UFR_3.30. The extractor splits on underscores and pulls out both pieces:

def _parse_country_code(code_string):
    parts = code_string.split("_")
    code = parts[0]
    if code == "EUR":
        code = "EU"  # the euro area block, not a country
    instrument_type = parts[4] if len(parts) > 4 else None  # SWP / GOV / OIS / PEE
    return code, instrument_type

Then, for every country column, it reads straight across rows 4–10 and assembles one row per country per curve type — this table is naturally wide (one row, several parameter columns) rather than long, since these are single scalars, not a repeating vector like the Qb data.

Note — on the no_VA sheet, row 10 (VA) is genuinely empty for every country — no volatility adjustment is applied there by definition, so this comes through as a null in the output rather than a parsing gap.
04

Saving to CSV without duplicating months

csv_store.py

Results are kept in two plain CSV files rather than a database — simplest thing that works, and easy to open in Excel or load into pandas/R directly. The one thing a flat file doesn't give you for free is idempotency: if you re-run November's file, naively appending would just double every row.

csv_store.py handles that by treating each (reference_date, curve_type) pair as a replaceable unit:

def _upsert_csv(csv_path, new_df):
    if csv_path.exists():
        existing = pd.read_csv(csv_path)
        keys = new_df[["reference_date", "curve_type"]].drop_duplicates()
        merge = existing.merge(keys, on=["reference_date", "curve_type"],
                                how="left", indicator=True)
        existing_kept = existing[merge["_merge"] == "left_only"]  # drop the old month
        combined = pd.concat([existing_kept, new_df], ignore_index=True)
    else:
        combined = new_df
    combined.to_csv(csv_path, index=False)  # rewrite the whole file
    return combined

In plain terms: before writing, it deletes any rows already on disk for the month(s) you're loading, then appends the freshly parsed rows. Run the same file five times and the CSV ends up exactly the same size each time — useful if EIOPA republishes a corrected file for a month you've already loaded.

05

Tying it together

main.py

This is the entry point. It accepts either one file pair, or a whole folder to process in one go:

python main.py --qb-file EIOPA_RFR_20241130_Qb_SW.xlsx \
               --ts-file EIOPA_RFR_20241130_Term_Structures.xlsx

# or, for a folder with many months of both files:
python main.py --input-dir /path/to/eiopa_monthly_downloads

For each file pair it: reads the reference date from both filenames and checks they agree (catching an accidental mismatch, e.g. pairing October's Qb file with November's Term Structures), runs both extractors, and saves each result via csv_store.py. In folder mode, it pairs files up automatically by matching the EIOPA_RFR_YYYYMMDD_ prefix and warns (rather than failing) if one side of a pair is missing.

Two bugs this caught along the way

Half‑year term indices — the first version of qb_extractor.py cast term indices to int. That silently merged 0.5→0 and 1.0→1 for the ten countries that use semi‑annual steps, dropping half their rows. Fixed by keeping term_index as a float end‑to‑end, including in the CSV.
openpyxl's read_only=True mode — it's built for reading rows in order, top to bottom. This pipeline instead jumps around a sheet by row and column number, and in read‑only mode that turned out to be quadratic — over 50 seconds just to scan one sheet's header row. Loading the workbook normally (no read_only flag) does the same job in under a second.

What ends up on disk

FileShapeColumns
data/qb_vectors.csvlongreference_date, curve_type, country, term_index, qb_value

Both are cumulative — every month you run adds its rows onto the same two files, so curve_parameters.csv naturally becomes a time series once you've run it for a few months in a row.

Running it yourself

Requires pandas and openpyxl (pip install pandas openpyxl). From inside the project folder:

python main.py --qb-file /path/to/EIOPA_RFR_20241130_Qb_SW.xlsx \
               --ts-file /path/to/EIOPA_RFR_20241130_Term_Structures.xlsx

Output lands in data/qb_vectors.csv and data/curve_parameters.csv, created automatically on first run.
